# 行为准则使用说明

本文说明如何在 App 中编写 **行为准则**、在阅读页 **记录今日践行**，以及 Markdown 写法与编辑器快捷按钮的对应关系。  
App 内路径：**设置 → 行为准则使用说明**。

## 功能概览（P1）

- 阅读页将笔记按 **块** 展示；部分块 **可点击**。
- 点击后记录 **今日遵守** 或 **今日违背**，可选备注；弹窗内可查看 **历史记录**。
- 块旁圆点：**绿色** = 今日遵守，**红色** = 今日违背，**浅灰** = 今日尚未记录。
- 践行记录保存在本地 `.meta/practice-logs.json`，**不会**随 GitHub 同步；笔记正文仍为 `.md` 文件。

## 哪些块可点击（记录践行）

| Markdown 写法 | 阅读页 | 编辑器快捷按钮 |
|---------------|--------|----------------|
| `> [!rule] 文字` | **可点击**，记录遵守/违背 | **准则** |
| `> [!habit] 文字` | **可点击**，记录遵守/违背 | **习惯** |

## 待办（不弹践行窗）

| Markdown 写法 | 说明 | 编辑器快捷按钮 |
|---------------|------|----------------|
| `- [ ] 文字` | 普通待办，仅展示勾选框，**不可点击**记录践行 | **待办** |
| `- [x] 文字` | 已完成待办 | 在编辑器中把 `[ ]` 改为 `[x]` |

普通段落、标题、`•` 列表项 **不可点击**（后续版本可能支持按空行分段自动识别）。

## 编辑器工具栏

在编辑笔记、键盘弹出时，底部工具栏提供：

- **待办**：当前行添加或去掉 `- [ ] `
- **准则**：当前行添加或去掉 `> [!rule] `
- **习惯**：当前行添加或去掉 `> [!habit] `

光标放在某一行（或选中多行）后点击即可，无需手打 `> [!rule]`。

## 示例笔记

```markdown
# 健康作息

> [!habit] 11 点睡觉，你没有晚睡的资本

- [ ] 睡前听听书，别刷手机

# 沟通

> [!rule] 别轻易跟旁边人诉苦，未必被理解，还可能引发矛盾

> [!rule] 行动才是解决焦虑的方法
```

## `- [ ]` 与 `- [x]` 的区别

二者都是 Markdown **待办** 语法，写在笔记文件里，会随 GitHub 同步：

| 写法 | 阅读页显示 | 常见含义 |
|------|------------|----------|
| `- [ ]` | ☐ 正文 | 清单上「未完成」 |
| `- [x]` | ☑ 正文（删除线） | 清单上「已完成 / 已内化」 |

**与「今日遵守 / 违背」不是一回事：**

| | 正文 `[ ]` / `[x]` | 践行记录（点块记录） |
|--|---------------------|----------------------|
| 存储 | 笔记 `.md` | 本地 `.meta/` |
| 表示 | 这条准则是否算「做完/内化」 | **今天** 有没有遵守或违背 |
| 修改 | 在编辑器里改 | 阅读页点击 **准则 Callout** → 遵守 / 违背 |

`- [ ]` / `- [x]` **不能** 点击记录践行；块旁绿/红点仅出现在 **准则 Callout** 上。

## `[!rule]` 与 `[!habit]` 的区别（写法习惯）

P1 中二者 **记录方式相同**（都是遵守 / 违背）。区别主要在 **你怎么分类准则**：

| 标记 | 建议用于 | 规划中的类型（P2 将细化交互） |
|------|----------|------------------------------|
| `[!habit]` | 每天可检的习惯（睡觉、戒刷手机） | daily |
| `[!rule]` | 遇到场景才用的原则（沟通、情绪、决策） | when |

## 一行一条准则（块与换行）

App 按 Markdown 文件里的 **换行符**（你在编辑器里按 **回车**）拆块，**不是**按屏幕上的显示行数。

| 写法 | 块数 | 能否点践行 |
|------|------|------------|
| 一行很长，阅读页自动折成多行显示 | **1 个** | ✅ |
| 准则中间按回车，续写行没有 `[!rule]` / `[!habit]` | **多个** | 仅首行 ✅ |
| 续写行写成 `> 补充说明`（无 `[!xxx]`） | **多个** | 仅首行 ✅ |

**建议：** 一条准则占 **一行**；文字可以很长，界面会自动折行，无需在准则中间手动换行。

```markdown
# 正确：一行，界面自动折行仍是一个块
> [!rule] 别轻易跟旁边人诉苦，未必被理解，还可能引发矛盾

# 不推荐：中间回车会变成两个块（第二行不可点践行）
> [!rule] 别轻易跟旁边人诉苦
未必被理解，还可能引发矛盾
```

只有带 `> [!rule]` 或 `> [!habit]` 的那一行才算准则 Callout；续行会当作普通段落展示。

## 践行记录如何绑定块（用户无感）

- 每个准则 Callout 在 App 内部分配 **隐式 ID**，存在 `.meta/block-registry.json`。
- **Markdown 里不会出现 ID**；若笔记里曾有 `^id` 后缀，保存时会自动去掉。
- 只改正准则文字时，践行圆点颜色 **不会丢**。

## 历史记录

- 点击准则块后，弹窗内 **「历史记录（N 条）」默认收起**；点击标题行展开/收起列表。
- 展开后展示日期时间、遵守/违背、备注，按时间倒序。
- 每次点「遵守」或「违背」都会 **追加一条**（带时间）；同一天可以有多条。
- 块旁圆点颜色取 **今日最新一条** 的结果。
- 「清除今日记录」在 **展开历史后** 列表下方（低调入口），需 **二次确认**；会删除该准则今天全部条目，过去日期保留。

## 数据与同步

| 内容 | 位置 | GitHub 同步 |
|------|------|-------------|
| 准则正文 | `notes/*.md` | 是 |
| 块隐式 ID | `.meta/block-registry.json` | 否（仅本机） |
| 今日遵守/违背 | `.meta/practice-logs.json` | 否（仅本机） |
