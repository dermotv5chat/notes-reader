# 行为准则使用说明

本文说明如何在 App 中编写 **行为准则**、在阅读页 **记录今日践行**，以及 Markdown 写法与编辑器快捷按钮的对应关系。  
App 内路径：**设置 → 行为准则使用说明**。

## 功能概览（P1）

- 阅读页将笔记按 **块** 展示；部分块 **可点击**。
- 点击后记录 **今日遵守** 或 **今日违背**，可选备注。
- 块旁圆点：**绿色** = 今日遵守，**红色** = 今日违背，**浅灰** = 今日尚未记录。
- 践行记录保存在本地 `.meta/practice-logs.json`，**不会**随 GitHub 同步；笔记正文仍为 `.md` 文件。

## 哪些块可点击

| Markdown 写法 | 含义 | 编辑器快捷按钮 |
|---------------|------|----------------|
| `- [ ] 文字` | 待办（未完成） | **待办** |
| `- [x] 文字` | 待办（已完成） | 在编辑器中把 `[ ]` 改为 `[x]` |
| `> [!rule] 文字` | 情境准则（推荐用于沟通、做事类原则） | **准则** |
| `> [!habit] 文字` | 习惯准则（推荐用于作息类） | **习惯** |

普通段落、标题、`•` 列表项 **暂不可点击**（后续版本可能支持按空行分段自动识别）。

## 编辑器工具栏

在编辑笔记、键盘弹出时，底部工具栏提供：

- **待办**：当前行添加或去掉 `- [ ] `
- **准则**：当前行添加或去掉 `> [!rule] `
- **习惯**：当前行添加或去掉 `> [!habit] `

光标放在某一行（或选中多行）后点击即可，无需手打 `> [!rule]`。

## 示例笔记

```markdown
# 健康作息

> [!habit] 11 点睡觉，你没有晚睡的资本

- [ ] 睡前听听书，别刷手机

# 沟通

> [!rule] 别轻易跟旁边人诉苦，未必被理解，还可能引发矛盾

> [!rule] 行动才是解决焦虑的方法
```

## `- [ ]` 与 `- [x]` 的区别

二者都是 Markdown **待办** 语法，写在笔记文件里，会随 GitHub 同步：

| 写法 | 阅读页显示 | 常见含义 |
|------|------------|----------|
| `- [ ]` | ☐ 正文 | 清单上「未完成」 |
| `- [x]` | ☑ 正文（删除线） | 清单上「已完成 / 已内化」 |

**与「今日遵守 / 违背」不是一回事：**

| | 正文 `[ ]` / `[x]` | 践行记录（点块记录） |
|--|---------------------|----------------------|
| 存储 | 笔记 `.md` | 本地 `.meta/` |
| 表示 | 这条准则是否算「做完/内化」 | **今天** 有没有遵守或违背 |
| 修改 | 在编辑器里改 | 阅读页点击块 → 遵守 / 违背 |

`- [ ]` 与 `- [x]` **都可以点击** 记录今日践行；块旁绿/红点来自践行记录，不是来自 checkbox 符号。

## `[!rule]` 与 `[!habit]` 的区别（写法习惯）

P1 中二者 **记录方式相同**（都是遵守 / 违背）。区别主要在 **你怎么分类准则**：

| 标记 | 建议用于 | 规划中的类型（P2 将细化交互） |
|------|----------|------------------------------|
| `[!habit]` | 每天可检的习惯（睡觉、戒刷手机） | daily |
| `[!rule]` | 遇到场景才用的原则（沟通、情绪、决策） | when |

## 可选：行末 `^id`

可在行末添加 Obsidian 式锚点，例如 `^sleep11`，便于改措辞后践行记录仍绑定同一条。  
**不强制**；不写时 App 会用行号与内容自动生成 ID。

```markdown
> [!habit] 11 点睡觉 ^sleep11
```

## 数据与同步

| 内容 | 位置 | GitHub 同步 |
|------|------|-------------|
| 准则正文 | `notes/*.md` | 是 |
| 今日遵守/违背 | `.meta/practice-logs.json` | 否（仅本机） |
